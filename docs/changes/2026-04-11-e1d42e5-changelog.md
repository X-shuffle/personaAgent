# Changelog（团队内部）— 2026-04-11 / e1d42e5

## Summary
本次提交为记忆检索增加“相似度阈值过滤”能力：可通过配置项控制最低相似度，低于阈值的记忆不再注入上下文，减少会话内召回噪声。

## Why
- 之前仅按 `topK` 召回，低相关记忆也可能进入提示词，影响回复稳定性。
- 需要一个可配置、可灰度的过滤开关，而不是把过滤逻辑写死在代码里。
- 该改动兼容现有 `inmem/qdrant` 两种存储模式，便于统一行为。

## Changed Files
- `.env.example`
  - 新增 `MEMORY_SIMILARITY_THRESHOLD=0` 示例配置。
- `internal/config/config.go`
  - `Config` 与 `envConfig` 新增 `MemorySimilarityThreshold float64`。
  - 从环境变量 `MEMORY_SIMILARITY_THRESHOLD` 读取值。
  - 在 `Load()` 中将阈值夹紧到 `[0,1]` 区间（小于 0 置 0，大于 1 置 1）。
- `cmd/server/main.go`
  - 构造记忆服务时，将 `cfg.MemorySimilarityThreshold` 透传到 `memory.NewService(...)`。
- `internal/memory/service.go`
  - `DefaultService` 新增 `minSimilarity` 字段。
  - `NewService(...)` 新增参数 `minSimilarity` 并做 `[0,1]` 夹紧。
  - `Retrieve(...)` 中增加 `m.Score < s.minSimilarity` 过滤逻辑。
- `internal/config/config_test.go`（新增）
  - 覆盖阈值读取测试：`0.77`。
  - 覆盖阈值夹紧测试：`-1 -> 0`、`2 -> 1`。
- `internal/memory/service_test.go`
  - 适配 `NewService` 新签名。
  - 新增相似度阈值行为测试：仅保留高于阈值的匹配项。

## Validation
- 配置层验证：`internal/config/config_test.go` 覆盖环境变量解析与边界夹紧。
- 检索层验证：`internal/memory/service_test.go` 覆盖阈值过滤行为。
- 回归影响面：`cmd/server/main.go` 仅增加参数透传，不改变存储后端选择分支。

## Risk / Notes
- 默认阈值为 `0`，线上默认行为与历史一致（仅不启用过滤）。
- 阈值配置过高可能导致“无记忆可召回”，建议从 `0.1~0.3` 小步调参。
- 当前阈值为全局配置，后续可考虑按会话/场景动态配置。
